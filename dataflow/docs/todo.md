## UI/UX 风格迭代任务清单 (对标 NocoDB)

### 1. 全局视觉 (Visual Foundation)
- [ ] **色彩系统重构**: 统一使用 Tailwind `slate`/`gray` 色系作为中性色，确立单一主色调 (Primary Blue/Indigo)，去除杂乱配色。
- [ ] **高密度排版**: 调整全局字体策略，正文倾向于 `text-sm` (13px/14px)，优化行高，提升一屏内的信息展示量。
- [ ] **容器卡片化**: 下拉菜单、模态框、卡片统一使用白色背景 + `border-gray-200` + 微阴影 (`shadow-sm`)。

### 2. 组件标准化 (Component Standardization)
- [ ] **Ghost 按钮推行**: 二级操作全部改为 Ghost/Text 模式（透明背景+灰色文字），仅主要 CTA 保留实心背景。
- [ ] **超级表格 (Hyper Grid)**: 改造数据展示表格，实现类似 NocoDB/Airtable 的视觉风格（细边框、Sticky Header、单元格选中态）。
- [ ] **统一工具栏**: 在各个模块顶部建立标准高度 (e.g., `h-10`) 的工具栏，容纳筛选、排序、视图切换等操作。

### 3. 核心模块优化 (Module Polish)
- [ ] **连接管理 (Connections)**: 列表项改为紧凑型卡片设计，增加环境 (Env) 颜色标签的视觉识别。
- [ ] **Dashboard**: 优化图表卡片的 Padding 和标题栏样式，使其更显轻量洁净。
- [ ] **SQL Editor**: 简化编辑器周边的边框和状态栏，强调代码区域的沉浸感。

### 4. 交互细节 (Interaction)
- [ ] **右键菜单全覆盖**: 将“右键操作”推广到连接列表、表对象管理、SQL 结果集等所有列表场景。
- [ ] **过渡动效**: 为 Hover、Modal 出现、路由切换增加统一的微交互动画 (`duration-200`)。
