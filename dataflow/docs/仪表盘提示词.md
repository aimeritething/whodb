  拓展后端开发提示词

  设计并实现仪表板管理系统的后端API，使用Node.js + TypeScript + Express：

  数据库设计：
  1. 主表设计：
     - dashboards: 仪表板基本信息
     - dashboard_components: 组件配置和位置
     - dashboard_templates: 模板数据
     - dashboard_versions: 版本历史
     - dashboard_shares: 分享配置
     - dashboard_comments: 评论数据

  2. 辅助表：
     - data_sources: 数据源配置
     - component_schemas: 组件配置模板
     - user_preferences: 用户偏好设置
     - theme_configs: 主题配置
     - audit_logs: 操作日志

  技术架构：
  1. 框架：Express.js + TypeScript
  2. ORM：Prisma或TypeORM
  3. 数据库：PostgreSQL + Redis缓存
  4. 认证：JWT + Passport.js
  5. 文件存储：MinIO或AWS S3
  6. 消息队列：Bull Queue
  7. API文档：Swagger/OpenAPI

  核心API设计：
  1. 仪表板管理 (/api/v1/dashboards)：
     - GET /: 列表（分页、搜索、筛选、排序）
     - POST /: 创建（支持从模板创建）
     - GET /:id: 详情（包含组件列表）
     - PUT /:id: 更新（基本信息、权限）
     - DELETE /:id: 删除（软删除）
     - POST /:id/duplicate: 复制仪表板
     - POST /:id/export: 导出功能

  2. 组件管理 (/api/v1/dashboards/:id/components)：
     - GET /: 获取组件列表
     - POST /: 添加组件
     - PUT /:componentId: 更新组件
     - DELETE /:componentId: 删除组件
     - POST /:componentId/duplicate: 复制组件
     - PUT /:componentId/position: 更新位置
     - POST /batch: 批量操作组件

  3. 数据管理 (/api/v1/data)：
     - GET /charts: 获取可用图表列表
     - GET /datasources: 数据源列表
     - POST /datasources: 创建数据源
     - POST /query: 执行数据查询
     - GET /preview: 数据预览

  4. 协作功能 (/api/v1/collaboration)：
     - POST /:id/share: 创建分享链接
     - GET /:id/permissions: 权限列表
     - PUT /:id/permissions: 更新权限
     - GET /:id/comments: 评论列表
     - POST /:id/comments: 添加评论
     - WebSocket: 实时协作

  高级功能API：
  1. 版本管理 (/api/v1/versions)：
     - GET /:id: 历史版本列表
     - POST /:id: 创建版本快照
     - POST /:id/restore: 恢复到指定版本
     - GET /:id/compare: 版本对比

  2. 模板管理 (/api/v1/templates)：
     - GET /: 模板列表（分类、标签）
     - POST /: 创建模板
     - POST /:id/use: 使用模板
     - POST /:id/favorite: 收藏模板

  3. 主题管理 (/api/v1/themes)：
     - GET /: 主题列表
     - POST /: 创建主题
     - GET /:id: 主题详情
     - PUT /:id: 更新主题

  性能优化：
  1. 数据库优化：
     - 索引优化（dashboard_id, created_at等）
     - 分页查询优化
     - 连接池配置

  数据库设计提示词

  设计完整的数据库表结构：

  ```sql
  -- 仪表板主表
  CREATE TABLE dashboards (
      id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
      name VARCHAR(255) NOT NULL,
      description TEXT,
      owner_id UUID NOT NULL,
      thumbnail_url VARCHAR(500),
      is_public BOOLEAN DEFAULT false,
      is_template BOOLEAN DEFAULT false,
      layout_config JSONB,
      theme_id UUID,
      created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
      updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
      deleted_at TIMESTAMP
  );

  -- 仪表板组件表
  CREATE TABLE dashboard_components (
      id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
      dashboard_id UUID NOT NULL,
      component_type VARCHAR(50) NOT NULL,
      name VARCHAR(255),
      position_x INTEGER DEFAULT 0,
      position_y INTEGER DEFAULT 0,
      width INTEGER DEFAULT 6,
      height INTEGER DEFAULT 4,
      config JSONB NOT NULL,
      data_config JSONB,
      style_config JSONB,
      is_locked BOOLEAN DEFAULT false,
      z_index INTEGER DEFAULT 1,
      created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
      updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
  );

  -- 数据源表
  CREATE TABLE data_sources (
      id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
      name VARCHAR(255) NOT NULL,
      type VARCHAR(50) NOT NULL,
      connection_config JSONB NOT NULL,
      refresh_interval INTEGER DEFAULT 300,
      is_active BOOLEAN DEFAULT true,
      created_by UUID NOT NULL,
      created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
      updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
  );

  -- 仪表板分享表
  CREATE TABLE dashboard_shares (
      id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
      dashboard_id UUID NOT NULL,
      shared_with_id UUID,
      share_token VARCHAR(255),
      permission_level VARCHAR(20) NOT NULL,
      expires_at TIMESTAMP,
      created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
  );

  -- 仪表板版本表
  CREATE TABLE dashboard_versions (
      id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
      dashboard_id UUID NOT NULL,
      version_number INTEGER NOT NULL,
      name VARCHAR(255),
      description TEXT,
      config_snapshot JSONB NOT NULL,
      created_by UUID NOT NULL,
      created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
  );

  -- 索引
  CREATE INDEX idx_dashboards_owner_id ON dashboards(owner_id);
  CREATE INDEX idx_dashboards_created_at ON dashboards(created_at);
  CREATE INDEX idx_components_dashboard_id ON dashboard_components(dashboard_id);
  CREATE INDEX idx_shares_dashboard_id ON dashboard_shares(dashboard_id);
  CREATE INDEX idx_versions_dashboard_id ON dashboard_versions(dashboard_id);
